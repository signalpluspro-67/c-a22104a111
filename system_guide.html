How The System Works

Methodology, not magic · generated 2026-07-21 · The Philosopher Investor
Most "backtested edges" are noise wearing a suit. This is the gauntlet every signal runs before it earns a dollar of size — and the honest scoreboard of what survived.

1 · How we separate edge from noise

A profitable backtest proves nothing — a coin-flipper gets lucky too. So every screener runs an eight-step statistical gauntlet. The hardest gate is the block bootstrap: it shuffles returns in 5-day blocks, so a signal that only looked good because a handful of winning days clustered together gets exposed as luck.

2 · Two engines: momentum vs mean-reversion

A stock is either trending or chopping, and the same trade loses in the wrong mode. So we run two engines. Momentum buys strength: PULLBACK enters a dip inside an uptrend (price > SMA50 > SMA200), BREAKOUT buys fresh highs out of a tight base. Mean-reversion buys panic: BOTTOM catches deep capitulation (≥ −40% off the 52-week high) with a reversal candle. Different math, different hold.

Lesson: a screener isn't "good" or "bad" in the abstract — it's good for the market state it was built for. Forcing a reversion trade in a runaway trend is how accounts bleed.

3 · Confluence: when two weak signals beat one strong

No single screener is a holy grail. The edge lives in agreement: when an oversold BOTTOM lines up with institutional options flow (SMART_FLOW) within a day, the combination is far more selective than either alone — fewer trades, higher conviction.
┌─ APPARENT SHARPE — SOLO VS STACKED ────────────────┐
│ BOTTOM solo          █████ 1.40                    │
│ BOTTOM x SMART_FLOW  ████████████████████████ 6.17 │
│                      0                   6.17      │
│                                                    │
│ -> tempting... but read the caveat below           │
└────────────────────────────────────────────────────┘
The honest part: stack two signals and the apparent Sharpe explodes past 6. We almost shipped that. Then the block bootstrap and the deflated Sharpe ratio (DSR went negative) said it plainly: with only ~71 trades, that 6.17 is clustered luck, not edge. Confluence genuinely sharpens selection — but the flashy number is a trap, and the discipline is to kill it anyway.

4 · Regime-aware: the same signal is a different trade

A dip-buy in a calm tape and a dip-buy in a VIX-30 panic are not the same bet. Sliced by volatility regime, the same screener's risk-adjusted return swings hard — strongest when the market is either very calm or genuinely stressed, weakest in the murky "normal" middle.
┌─ SAME SIGNAL, SHARPE BY VIX REGIME ─────┐
│ CALM      ██████████████ 2.14           │
│ NORMAL    ██████ 0.94                   │
│ ELEVATED  ████████████ 1.84             │
│ STRESS    ████████████████████████ 3.64 │
│           0                   3.64      │
│                                         │
│ -> calm vs stress are different worlds  │
└─────────────────────────────────────────┘
How we use it: we tried to turn this into a hard "skip elevated VIX" filter — it only helped marginally (≈ +0.3 Sharpe), so regime stays context, not a trigger. It tells us how much to size and what to expect, not whether to pull the trigger.

6 · The Kerberos story

A lot of this system started by reverse-engineering @kerberos007, a cult quant whose threads describe momentum factors, regression-slope ranking, and breadth "Jaws" divergences. We rebuilt them — and then did the thing the threads never do: ran each one through the same block bootstrap.

What we learned: most of the famous narrative signals were already built here, and the rigorous ones (a Clenow-style momentum factor, breadth/Jaws indicators) stand as context; the flashy "parabolic" and overlay tricks did not survive. Borrowing a smart trader's ideas is cheap. Auditing them honestly is the actual edge.
